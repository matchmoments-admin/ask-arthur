Ask Arthur — Ingestion + Voyage Embedding Flows

Where scam-data enters the system · Sonnet 4.6 classification · Voyage 3.5 embedding lanes · cost-telemetry tags · snapshot 2026-05-17

Seven ingestion lanes

1. User submission → /api/analyze → scam_reports · 2. Bot dispatch (5 platforms, pg_net webhook) → shared scam_reports · 3. Narrative scrapers (scamwatch/acsc/asic/austrac) → feed_items · 4. Inbound email (Cloudflare → Worker → Edge Fn) → feed_items · 5. Reddit (classify→embed→cluster) → reddit_post_intel + themes · 6. ACNC charity register (chunked daily) → acnc_charities + sibling embeddings · 7. IOC feeds (12 scrapers) → vulnerability_iocs (no embed)

Voyage 3.5 writers (5 Inngest functions)

scam-report-embed (event-driven on store) · feed-items-embed (cron 30m, narrative only) · reddit-intel-embed (event after Sonnet classify) · acnc-charity-backfill-embed (nightly 04:00 + manual) · backfill-embed (manual ops). All log cost_telemetry with feature-specific tags. Two retrieval-side Voyage calls (getSimilarReports, getRelevantThemes) currently un-logged — P3 fix.

Sonnet 4.6 classifier (Reddit only)

Single Sonnet call per batch (≤40 posts) emits per-post intent_label · modus_operandi · brands_impersonated · victim_emotion · novelty_signals · tactic_tags · scrubbed quotes plus daily aggregate (lead_narrative + emerging_threats + brand_watchlist). ~$0.20/batch, cap A$10/day. Retry-with-feedback on Zod-validation failure (PR #228).

Sibling-index compliance

Reference pair: acnc_charities (daily UPDATE, no embedding column) ↔ acnc_charity_embeddings (HNSW 1024d Voyage, v121 split, v122 dropped parent vec cols). Partial-HNSW exceptions allowed: scam_reports.embedding WHERE embedding IS NOT NULL (no reindex on NULL→populated transition) and feed_items.embedding partial IVFFlat (narrative only). Born from incident 2026-05-09. ADR-0005.
