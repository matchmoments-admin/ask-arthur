# Stack pins — Ask Arthur installed versions

When a grilling session (or any work) touches one of the libraries below, the model's training-data defaults frequently drift from the installed version. Read the relevant section before writing or reviewing code that uses these APIs.

Generated by inspection of `package.json` files in the monorepo. Update this file when versions bump.

---

## React 19

**Installed in:** `apps/web`, `apps/extension`, `apps/mobile`.

**Drift hazards:**

- `useFormState` is renamed to `useActionState` (`react`, not `react-dom`).
- `useFormStatus` lives in `react-dom`.
- The new `use(...)` hook unwraps thenables and contexts; it does not replace `useEffect` for data fetching but is heavily used with React Server Components.
- `forwardRef` is no longer required for function components — `ref` is a regular prop.
- The compiler is opt-in; the codebase does **not** currently enable it.

---

## Next.js 16 (Turbopack stable)

**Installed in:** `apps/web`.

**Drift hazards:**

- `cookies()`, `headers()`, `draftMode()`, `params`, and `searchParams` are **async**. Always `await` them. `const cookieStore = await cookies()`.
- Turbopack is the default dev + build bundler. Webpack-specific config (`webpack.config.js`, `webpack` block in `next.config.ts`) is a regression signal.
- `next/dynamic` with `ssr: false` is **forbidden inside Server Components**. Mark the wrapper `"use client"` first.
- Middleware runtime: `nodejs` (default) or `edge` — pick explicitly in `apps/web/middleware.ts`. The 25s Vercel cap applies; auth calls must use `Promise.race` with a 3s budget (incident 2026-05-09).
- App Router file conventions: `route.ts`, `page.tsx`, `layout.tsx`, `loading.tsx`, `error.tsx`, `not-found.tsx`. Pages Router is not present here.
- Server Actions use the `"use server"` directive; they cannot be called from middleware.

## Zod 4

**Installed across the monorepo via `@askarthur/types`.**

**Drift hazards:**

- `z.string().url()`, `z.string().email()`, `z.string().uuid()`, `z.string().ip()` are **removed**. Use the top-level `z.url()`, `z.email()`, `z.uuid()`, `z.ipv4()`/`z.ipv6()` instead.
- `.passthrough()` / `.strict()` / `.strip()` API has changed; the v4 default is `strip`. Prefer `z.object({...}).passthrough()` style explicitly when needed.
- `ZodError.errors` → `ZodError.issues`. Error inspection code that walks `.errors` was correct in v3, breaks in v4.
- `z.discriminatedUnion` no longer requires the second `[...]` array argument in v4-style call; check the schema file's style before mimicking.

## Expo 54 (React Native)

**Installed in:** `apps/mobile`.

**Drift hazards:**

- New Architecture (Fabric + TurboModules) is on by default. Native modules that don't support it must be opted out per-module.
- Expo Router v5 file-based routing — `app/` directory inside `apps/mobile/`. Distinct from Next.js's `app/`.
- `expo-image` replaces `react-native-fast-image`; `<Image />` from `expo-image` not `react-native`.
- `expo-modules-core` is the bridge for custom native code.
- EAS Build profiles in `eas.json` — `development`, `preview`, `production` are the only supported names in this repo.

## WXT (cross-browser extension framework)

**Installed in:** `apps/extension`.

**Drift hazards:**

- Manifest V3 only. Background scripts use the `service_worker` entry, not persistent background pages.
- Entrypoints are convention-based: `entrypoints/background.ts`, `entrypoints/content.ts`, `entrypoints/popup/index.html`. Don't author `manifest.json` directly — WXT generates it.
- `WXT_FACEBOOK_ADS=true` toggles the optional Facebook ad-scan host permissions for the full-featured v1.0.1 release (see root CLAUDE.md Deployment table).
- Cross-browser via `wxt build` + `wxt zip`. Firefox builds need a different signing flow than Chrome — handled per-target by WXT, not manually.

## Supabase JS v2

**Installed in:** every `apps/*` and most `packages/*`.

**Drift hazards:**

- `auth.getSession()` reads from cookie only — does NOT refresh. Use `auth.getUser()` for a fresh validation (network call) — but wrap in `Promise.race` per the incident 2026-05-09 rule.
- `from('table').select()` returns `{ data, error }` — never throws. Always destructure and check `error` before using `data`.
- RLS is on by default for new tables — explicitly `CREATE POLICY` per [`supabase/CLAUDE.md`](../../../supabase/CLAUDE.md).
- The three client factories (`createServerClient`, `createBrowserClient`, `createServiceClient`) come from `@askarthur/supabase`, not `@supabase/supabase-js` directly. See [`packages/supabase/CLAUDE.md`](../../../packages/supabase/CLAUDE.md).
- `supabase.functions.invoke()` is for edge functions; for RPCs use `supabase.rpc('name', args)`.

## Inngest SDK (current major)

**Installed in:** `packages/scam-engine`, `apps/web`.

**Drift hazards:**

- `inngest.createFunction({ id, name }, { event }, handler)` — `id` must be stable across deploys. Renaming the `id` orphans in-flight runs. Coordinate via feature flag.
- `step.run('label', async () => ...)` results are checkpointed; never put side-effects outside a `step.run`.
- `step.sleepUntil` is precise to the second; `step.sleep` is best-effort.
- The local dev server runs at `http://127.0.0.1:8288`; it exposes an MCP endpoint at `/mcp` when present.
- Cost telemetry pattern: `step.run('cost', () => logCost({...}))` — `logCost` lives in `apps/web/lib/cost-telemetry.ts`; scam-engine packages duplicate the `PRICING` constants to avoid an apps/web dep cycle (see `packages/scam-engine/src/inngest/analyze-cost.ts`).

## Usage from the skill

When a session touches code that imports any of the above libraries, the grilling session should:

1. Pull up this file first.
2. Cross-check the user's plan / code against the drift hazards.
3. If the user states something that conflicts with a hazard (e.g. "I'll call `cookies()` synchronously"), challenge it immediately with a pointer to the hazard.
4. If you discover the version has bumped (check `package.json`), update this file inline as part of the session.
